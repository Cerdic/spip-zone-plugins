Formulaire pour spip_meta

Le même formulaire CFG sans Ajax

On va utiliser le meme fond CFG mais toute la page sera rechargée, donc pas uniquement le formulaire.

Code :
#CFG_TRAITER{#ENV{cfg_vue}}
#CFG_ARBO{exemple_meta}
#FORMULAIRE_CFG{exemple_meta,'','',non}

#CFG_TRAITER{#ENV{cfg_vue}} #CFG_ARBO{exemple_meta} #FORMULAIRE_CFG{exemple_meta,'','',non}

Lorsque l'on valide, toute la page se recharge et la balise #CFG_ARBO avant la balise #FORMULAIRE_CFG contient les modifications apportées.

Il faut néanmoins la balise #CFG_TRAITER avant le code susceptible d'être affecté par les modifications sinon le traitement ne se fait pas.

Il faudrait trouver une autre écriture pour cela qui puisse faire le traitement depuis cfg_fonctions.php par exemple si le formulaire est posté en dehors d'ajax et en dehors de ?exec=cfg ce qui est problématique vu que le formulaire cfg ajoute systématiquement le paramètre exec=cfg... et il sera difficile de faire afficher les méssages d'erreur de cette façon.
