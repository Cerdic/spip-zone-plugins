Gestion des nouveaux champs dans les tables SPIP par les champs EXTRA : champs homonymes

Plugin Champs homonymes: visualiser le résultat

Afficher le résultat de la synchronisation

#DEBUT_SURLIGNE

Cette page vous permet de visualiser le résultat de la synchronisation des champs EXTRA avec leurs homonymes


Ce qui a été fait

  • L'étape précédente a permis d'ajouter à plusieurs table de SPIP un nouveau champs nommé "plus".
  • Dans le fichier "mes_options_homonymes.php" contenu dans le dossier du plugin nous avons défini des champs EXTRA nommé "plus".
  • Dans le fichier "mes_fonctions_homonymes.php" contenu dans le dossier du plugin nous avons défini une fonction nommée "extra_homonyme_maj" qui nous permettra de synchroniser nous champs.

Principe

Dans cette page, pour chaque table, nous allons commencé par la synchronisation:
‹BOUCLE_articles1(ARTICLES)>
[(#ID_ARTICLE|extra_homonyme_maj{'articles',#ENV{homonyme}})]
‹/BOUCLE_articles1>

Selon le choix que vous ferez dans cette page, la valeur de #ENV{homonyme} demandera à la fonction de synchroniser le nouveau champs "plus" avec la valeur contenue dans le champ EXTRA "plus" ou vise-versa.

Dans la boucle suivante, nous afficherons cette valeur par la balise [(#EXTRA|extra{plus})] pour la valeur contenue dans le champ EXTRA "plus" et par la balise [(#PLUS)] pour la valeur contenue dans le nouveau champ homonyme.

‹BOUCLE_articles(ARTICLES)>
[‹li class="spip">
‹a href="ecrire/?exec=articles&id_article=#ID_ARTICLE" target="_blank" class="spip_out">TITRE de l'ARTICLE: #TITRE‹/a>
‹div style='text-align: left;' class='spip_code' dir='ltr'>
‹ul>
‹li>EXTRA plus = (#EXTRA|extra{plus})‹/li>
[‹li>Plus = (#PLUS)‹/li>]
‹/ul>
‹/div>
‹/li>]
‹/BOUCLE_articles>

Entrer des valeurs par l'interface privée

Dans l'interface privée,
  • modifiez une rubrique, un article, une brève, un mot clé ou un site référencé pour donner une valeur au champ EXTRA plus.
  • rafraichîssez cette page pour voir le résultat s'afficher

Supprimer les nouveaux champs

Maintenant que vous avez pu tester le bon fonctionnement du plugin, vous pouvez supprimer les champs "plus" que nous avons créés pour le test.
