Interface pour CFG 2.x
----------------------

CFG 2.x fournit 3 choses :
- la gestion de #FORMULAIRE_XX utilisant des parametres CFG
- les fonctions (lire|ecrire|effacer)_config et une modification de #CONFIG

CFG 2.x ne fournit plus la gestion des pages ?exec=cfg. C'est le rôle
de ce plugin CFG Interface.

----------------------

Ce plugin

- liste des fonds CFG présents dans le dossier /cfg sur la page ?exec=cfg
  (todo)

- charge un fond particulier si on passe ?exec=cfg&cfg=xx :
  * cherche : cfg/cfg_xx.html
  * sinon : formulaires/config_xx.html
  * titre automatique : <:xx:configuration_xx:>
  * descriptif automatique : <:xx:description_xx:>
  * menu automatique : cfg/nav-config_xx.html sinon cfg/nav-config_defaut.html
  
- charge un onglet particulier si on passe ?exec=cfg&cfg=xx&onglet=yy :
  * cherche : cfg/cfg_xx_yy.html
  * sinon cfg/cfg_xx.html
  * sinon formulaires/config_xx_yy.html
  * titre automatique : <:xx:configuration_xx_yy:> sinon <:xx:configuration_xx:>
  * descriptif automatique :<:xx:description_xx_yy:> sinon <:xx:description_xx:>
  * menu automatique : cfg/nav-config_xx.html sinon cfg/nav-config_defaut.html

-----------------------

Cas particulier :
- si un fonds/cfg_xx.html existe alors ce plugin charge le plugin de compatibilité
  avec les anciens fonds CFG. Si le plugin n'est pas installé, un message l'indique.


-----------------------

En résumé :

  * Utilisation du l'ecriture simplifiee prive/exec/cfg.html (SPIP 2.0.6+)
  
  * exec=cfg&cfg=nom cherche une page fonds/cfg_nom.html et la charge
  sinon appelle directement #FORMULAIRE_CONFIG_$NOM

  * Les sous éléments de configuration peuvent s'appeler par
  exec=cfg&cfg=nom&onglet=element et cherchera une page fonds/cfg_config_nom_element.html,
  sinon fonds/cfg_config_nom.html, sinon appelle directement #FORMULAIRE_CONFIG_$NOM_$ELEMENT
