Contacts d'une organisation, et liaisons

Pour lister les contacts d'une organisation :

<BOUCLE_c(CONTACTS){id_organisation}{par nom}>
	\#CIVILITE
	\#PRENOM
	\#NOM
	\#ID_CONTACT
	...

Pour lister les liaisons d'un contact :

<BOUCLE_cl(CONTACTS_LIENS){id_contact}{par objet, id_objet}>
	\#OBJET
	\#ID_OBJET
	\#TYPE_LIAISON
  • #NOM, #ID_ORGANISATION (auteur: #ID_AUTEUR)
    • #NOM #PRENOM, #ID_CONTACT
      • ( #OBJET, #ID_OBJET, #TYPE_LIAISON )
        • [(#PAGINATION|?{
      • …
        • })]
      • [(#PAGINATION|?{
    • …
      • })]
    • [(#PAGINATION|?{
  • …
    • })]

Auteurs - contacts

Pour lister les auteurs - contact (auteurs liés à un contact)

<BOUCLE_a(AUTEURS){id_contact>0}>
	\#NOM
	...

Pour utiliser les balises de la table contacts comme \#PRENOM, il faut ajouter la table à la boucle. Cependant, on ne peut pas utiliser des champs homonymes à la table auteur.

<BOUCLE_a(AUTEURS contacts){id_contact>0}>
	\#NOM
	\#PRENOM (table contacts)
	...
    0}{pagination 3}>
  • #LOGIN / #NOM / id_contact: [(#ID_CONTACT)], prenom: [(#PRENOM)]
    • [(#PAGINATION|?{
  • …
    • })]

Contacts - auteurs

Pour lister les contacts - auteurs (contacts liés à un auteur)

<BOUCLE_a(CONTACTS){id_auteur>0}>
	\#NOM
	...

Pour utiliser les balises de la table auteurs comme \#LOGIN, il faut ajouter la table à la boucle. Cependant, on ne peut pas utiliser des champs homonymes à la table contacts.

<BOUCLE_a(CONTACTS auteurs){id_auteur>0}>
	\#NOM
	\#LOGIN (table auteurs)
	...
    0}>
  • #ID_CONTACT / #NOM / #PRENOM / id_auteur: [(#ID_AUTEUR)], login: [(#LOGIN)]
