Ce fichier texte traitera des petites modifications a faire sur spip pour amèliorer l'anonymat des visiteurs de votre site.

L'adresse IP d'un ordinateur peut permettre à un organisme tiers de définir précisément qui et à quelle heure à poster tel message sur tel site. Il faut donc tout faire pour un site sensible afin d'éviter que ces adresse IP puissent-êtres collecter par ces organismes.
La première chose à faire est donc d'éviter que le site que vous avez mis en place stocke ces adresses IP, car en cas de saisie (par la justice), elle constituerons autant de preuve de la visite de tel ou tel personnes.


1 - Le stockage des adresses IP

1.1 - Les forums

Lorsque vous autorisez les forums sous les articles ou les brèves, les visiteurs de votre site peuvent laisser un petit message qui sera affiché sur le site. Or, en plus d'enregistrer le contenu du message est éventuellement un pseudo et une adresse email, spip enregistre aussi l'adresse IP de la personne.

1.2 - Les statistiques

Spip utilise un système de statistique se basant sur le nombre d'adresses IP différentes. Cela peut-être utile pour déterminer avec précision l'audiance de votre site, mais constitue à l'instar des forums un trou de sécurité qu'il convient de combler.

1.3 - Solution

Pour éviter de stocker les adresses IP, il suffit d'ajouter dans le répertoire "ecrire/" un fichier "mes_options.php" contenant la ligne suivante : "$ip = substr(md5($ip),0,16);"

1.4 - Exemple et documentation de spip

// Pour renforcer la privacy, decommentez la ligne ci-dessous (ou recopiez-la
// dans le fichier ecrire/mes_options : SPIP ne pourra alors conserver aucun
// numero IP, ni temporairement lors des visites (pour gerer les statistiques
// ou dans spip.log), ni dans les forums (responsabilite)
$ip = substr(md5($ip),0,16);
